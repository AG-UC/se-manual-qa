SE-3091 — Cookiebot YSC smoke

Static test page for the scan-adapter cookie-purpose cache-aliasing fix (Jira: SE-3091 / CTS-4295). Loads YouTube so the Cookiebot scanner observes YSC (and VISITOR_INFO1_LIVE) being set; the post-scan cookie declaration should list these cookies with a non-empty CookiePurposes field in English.

Smoke target: after a Cookiebot scan of this page, the cookie declaration for the test CBID must list YSC with a non-empty purpose string (e.g. "Registers a unique ID to keep statistics of what videos from YouTube the user has seen."). Pre-fix: purpose may be empty when the YSC cache key was last written by another tracker. Post-fix: the immutable record contract guarantees the purpose survives.

What's on this page

One embedded YouTube player. YouTube drops YSC (session-id, statistics category) and VISITOR_INFO1_LIVE on play / load; the Cookiebot scanner observes both as DocumentCookie entries, which is enough to exercise the same-scan / same-cache-key path the fix repairs.

Out-of-scope on this page

  • No UC CMP. Cookiebot customers inject their own banner from the Cookiebot CDN once a CBID is bound to this URL.
  • No GA / GTM / Meta / LinkedIn / HubSpot / DoubleClick. Their dev-DB purposes are incomplete; including them would muddy the smoke signal. For a broader translation matrix verification, do that as a production retest after redeploy + rescan.
  • No anchor links to sibling QA pages. Self-contained — sibling links cause the scanner to crawl outward and contaminate signals.
